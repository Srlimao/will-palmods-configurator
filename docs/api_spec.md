# ModConfigurator: Server & Website API Specification

This document details the API endpoints, payload structures, and query parameters required for communication between the in-game Lua mod, the React web configurator, and the backend server.

---

## 1. Web Configurator Redirect URL

When the player selects a client mod (e.g. `HUDLocator`) in the configuration menu and clicks **"Configure"**, the client mod opens the browser to the following URL:

```
https://pal-mod-configurator.dunhas.com/?userId=<UserId>&mod=<ModName>&config=<encoded_json>
```

### URL Query Parameters:
*   `userId` (string): The unique hexadecimal token or Steam ID of the player.
*   `mod` (string): The identifier of the mod being configured (e.g. `HUDLocator` or `AccessoryToggler`).
*   `config` (string): The URL-encoded JSON string representing the player's current local settings.

*The React frontend should check these URL parameters on load to populate its forms with the current config and establish the active user session.*

---

## 2. Retrieve Configuration API (In-Game Mod -> Backend Server)

When the player clicks **"Sync Configs"** in the mod overlay menu, the in-game background daemon will request the latest saved config via a `GET` request.

### Request Details:
*   **Method**: `GET`
*   **Path**: `/api/config/get`
*   **Query Parameters**:
    *   `userId=76561197960287930` (Player's SteamID64 or unique ID)
    *   `mod=HUDLocator`

### Expected Responses:
*   **Success (200 OK)**: Returns the configuration JSON and a cryptographic `writeKey` generated by the server:
    ```json
    {
      "config": {
        "Global": {
          "Enabled": true,
          "Language": "en",
          "ScanIntervalMs": 1500
        },
        "Players": {
          "Enabled": true,
          "MaxDistance": 15000.0,
          "Style": {
            "DrawBox": false,
            "FontScale": 1.2
          }
        }
      },
      "writeKey": "8a3cf8d4b9b6e8...8f1a"
    }
    ```
    *Note: The game client must store this `writeKey` locally inside the mod configuration directory to authorize future in-game settings uploads.*

*   **Not Found (404 Not Found)**:
    ```json
    {
      "success": false,
      "error": "No configuration found for this User ID and Mod."
    }
    ```

## 3. Save Configuration API

This endpoint accepts two modes of authentication: Cookie Session Auth (for the React web interface) or Write-Key Auth (for the headless game client).

### Mode A: Web Browser Interface (Cookie Session Auth)
When the user edits their settings on the React website, the browser automatically sends the secure cookie session JWT (`token`) along with the request.

*   **Method**: `POST`
*   **Path**: `/api/config/save`
*   **Headers**: 
    *   `Content-Type: application/json`
*   **Body Payload**:
    ```json
    {
      "mod": "HUDLocator",
      "config": {
        "Global": { "Enabled": true, "Language": "en" }
      }
    }
    ```
    *(The backend server automatically extracts the Steam ID from the cookie JWT session)*

---

### Mode B: In-Game Client Interface (Write-Key Signature Auth)
When the user changes settings in the in-game menu, the headless cURL daemon POSTs the new configuration, verifying the write using the stored cryptographic key.

*   **Method**: `POST`
*   **Path**: `/api/config/save`
*   **Headers**: 
    *   `Content-Type: application/json`
*   **Body Payload**:
    ```json
    {
      "userId": "76561197960287930",
      "mod": "HUDLocator",
      "writeKey": "8a3cf8d4b9b6e8...8f1a",
      "config": {
        "Global": { "Enabled": true, "Language": "en" }
      }
    }
    ```
    *(The backend server validates the `writeKey` by computing a SHA256 HMAC of the `userId` using a server secret. If the calculated signature matches the request `writeKey`, the save is authorized).*

### Expected Responses (Both Modes):
*   **Success (200 OK)**:
    ```json
    {
      "success": true,
      "message": "Configuration saved successfully."
    }
    ```
*   **Validation / Auth Failure (401 Unauthorized)**:
    ```json
    {
      "success": false,
      "error": "Unauthorized: Invalid write key or missing token session."
    }
    ```
